# SSM Desktop Automatic Updates

SSM Desktop uses `electron-updater` with GitHub Releases as the update source.

This update system is Electron-only. It does not touch race generation, timing, beam break handling, meet workflows, SSL APIs, or database behavior.

## Update Source

GitHub Releases repository:

```text
Lee22bird/speedskate-meet
```

The packaged app reads update metadata from the GitHub release assets generated by Electron Builder.

## Channels

Supported channels:

- `alpha`
- `beta`
- `stable`

The app chooses a channel in this order:

1. `SSM_UPDATE_CHANNEL`
2. `UPDATE_CHANNEL`
3. Version suffix:
   - `0.1.0-alpha` -> `alpha`
   - `0.1.0-beta` -> `beta`
   - no suffix -> `stable`

For packaged alpha/beta builds, `electron-updater` allows prerelease updates. Stable builds stay on stable releases.

## Runtime Behavior

On packaged desktop startup:

1. SSM checks GitHub Releases for updates.
2. SSM checks again every 4 hours.
3. Updates download silently in the background.
4. When the update is downloaded, SSM shows:

   ```text
   A new version of SSM Desktop is ready.
   [Install Now] [Later]
   ```

5. `Install Now` restarts and installs immediately.
6. `Later` keeps the update staged and installs it the next time SSM Desktop restarts.

Development builds do not check for updates unless explicitly enabled:

```sh
SSM_ENABLE_AUTO_UPDATES=1 npm run desktop:dev
```

To disable updates in a packaged build:

```sh
SSM_DISABLE_AUTO_UPDATES=1
```

## Release Assets Required

Each GitHub Release must include the generated macOS assets from `release/mac/`, including:

```text
SSM Desktop.dmg
SSM Desktop.dmg.blockmap
SpeedSkateMeet-<version>-arm64-mac.zip
SpeedSkateMeet-<version>-arm64-mac.zip.blockmap
alpha-mac.yml / beta-mac.yml / latest-mac.yml
```

Electron Builder creates channel metadata based on the app version and channel settings.

## Publishing

Build and notarize:

```sh
npm run release:mac
```

Upload the release artifacts to GitHub Releases manually, or publish with Electron Builder using a GitHub token:

```sh
export GH_TOKEN="github_pat_or_token"
npx electron-builder --config electron-builder.yml --config.directories.output=/private/tmp/ssm-desktop-release/mac --mac --arm64 --publish always
```

Use GitHub prereleases for alpha and beta channels.

## Future Windows Support

The update service is platform-neutral, but Windows packaging is not enabled yet.

Before turning on Windows updates:

- Add a Windows target such as NSIS.
- Configure Windows code signing.
- Publish Windows installer assets and update metadata to the same GitHub Releases flow.
- Verify install/restart behavior on Windows.
